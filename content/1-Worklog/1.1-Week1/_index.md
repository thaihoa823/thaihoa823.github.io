### Week 1 Achievements:

* Gained a basic understanding of AWS and the role of cloud computing in providing technology resources on demand.

* Understood several important benefits of cloud computing:
  * Rapid resource deployment
  * Flexible system scaling
  * Reduced initial infrastructure investment
  * Payment based on actual usage

* Identified the main components of AWS Global Infrastructure:
  * AWS Region
  * Availability Zone
  * Edge Location

* Gained an overview of the main AWS service categories:
  * Compute
  * Storage
  * Database
  * Networking
  * Security

* Learned how to access the AWS Management Console, change the active Region, and search for AWS services.

* Understood that AWS Free Tier allows new users to practice with selected services within specific usage limits, while requiring careful monitoring to avoid unexpected charges.

* Became familiar with Cost Explorer and AWS Budgets for monitoring, analyzing, and controlling account costs.

* Learned how to create a budget and configure email notifications when spending reaches a specified threshold.

* Gained a basic understanding of the AWS Shared Responsibility Model:
  * AWS is responsible for securing the underlying cloud infrastructure
  * Customers are responsible for protecting their data, accounts, and resource configurations

* Completed the installation of AWS CLI and used a version-checking command to confirm that the tool was working correctly.

* Identified the functions of the main AWS IAM components:
  * User represents a specific individual or account
  * Group helps manage permissions for multiple Users
  * Role provides temporary access permissions
  * Policy defines which actions are allowed or denied

* Understood the principle of least privilege, in which users and services are granted only the permissions required to perform their assigned tasks.

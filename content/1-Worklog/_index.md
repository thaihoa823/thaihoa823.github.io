This section presents my weekly worklog during the First Cloud AI Journey internship program.

The internship lasted for **12 weeks**, from **April 15, 2026, to July 3, 2026**. During this period, I learned fundamental AWS concepts, practiced using several AWS services, recorded the results of each exercise, and reviewed the resources created during the labs.

The worklog was completed gradually each week. I followed the provided workshop materials, performed basic practical exercises through the AWS Management Console and AWS CLI, checked the results, documented the knowledge gained, and removed temporary resources when they were no longer required.

The main learning activities for each week are listed below:

**Week 1:** [Introduction to AWS, AWS Free Tier, account management, and basic cost awareness](1.1-week1/)

**Week 2:** [Learning AWS Identity and Access Management, including users, groups, roles, policies, and permissions](1.2-week2/)

**Week 3:** [Learning Amazon VPC fundamentals and reviewing AWS Site-to-Site VPN architecture](1.3-week3/)

**Week 4:** [Practicing basic Amazon EC2 configuration, instance access, Security Groups, and instance management](1.4-week4/)

**Week 5:** [Learning Amazon RDS fundamentals, database creation, connectivity, monitoring, and resource cleanup](1.5-week5/)

**Week 6:** [Learning Elastic Load Balancing, EC2 Auto Scaling, Launch Templates, Target Groups, and scaling policies](1.6-week6/)

**Week 7:** [Reviewing AWS Billing and Cost Management, AWS Budgets, cost monitoring, and billing alerts](1.7-week7/)

**Week 8:** [Learning Amazon CloudWatch Metrics, Logs, Logs Insights, Alarms, Metric Filters, and Dashboards](1.8-week8/)

**Week 9:** [Learning how to use AWS Support Center, create support cases, and understand support plans and severity levels](1.9-week9/)

**Week 10:** [Reviewing hybrid DNS architecture using Route 53 Resolver, AWS Managed Microsoft AD, and AWS CloudFormation](1.10-week10/)

**Week 11:** [Installing and configuring AWS CLI and practicing basic commands with S3, IAM, VPC, and EC2](1.11-week11/)

**Week 12:** [Learning AWS IAM Identity Center, AWS Organizations, users, groups, permission sets, account assignments, and SSO access](1.12-week12/)

Through these twelve weeks, I improved my understanding of AWS services, cloud security, resource monitoring, cost management, command-line operations, and centralized access management. The weekly worklog also helped me develop better technical documentation, self-learning, troubleshooting, and time-management skills.

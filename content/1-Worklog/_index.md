This section presents my weekly worklog during the First Cloud AI Journey internship program.

The internship lasted for **12 weeks**, from **April 17, 2026, to July 20, 2026**. During this period, I studied fundamental AWS concepts, practiced using various AWS services, reviewed system architectures, recorded the results of each exercise, and managed the resources created during the practical labs.

The worklog was developed and updated throughout the internship. Each week, I followed the provided workshop materials, completed practical exercises through the AWS Management Console and AWS CLI, verified the results, documented the knowledge and skills gained, and removed temporary resources when they were no longer needed.

The main learning activities completed each week are listed below:

**Week 1:** [Learning AWS fundamentals, global infrastructure, the AWS Management Console, Free Tier, cost management, AWS CLI, and IAM](1.1-week1/)

**Week 2:** [Learning AWS networking services, including Route 53, CloudFront, Elastic Load Balancing, VPC Peering, Site-to-Site VPN, Transit Gateway, Security Groups, Network ACLs, AWS WAF, and network monitoring](1.2-week2/)

**Week 3:** [Learning AWS compute and storage services, including Amazon EC2, Lightsail, containers, Amazon ECS, Amazon EKS, AWS Lambda, and Amazon S3](1.3-week3/)

**Week 4:** [Learning AWS storage services, including Amazon S3 storage classes, S3 Versioning, Amazon EBS, Snapshots, Amazon EFS, Amazon FSx, Storage Gateway, and AWS Backup](1.4-week4/)

**Week 5:** [Learning AWS database services, including Amazon RDS, Amazon Aurora, DynamoDB, ElastiCache, and Amazon Redshift](1.5-week5/)

**Week 6:** [Learning data lake and ETL concepts using Amazon S3, AWS Glue, Amazon Athena, AWS Lake Formation, and Amazon Macie](1.6-week6/)

**Week 7:** [Learning AWS security services and principles, including least privilege, GuardDuty, Security Hub, Amazon Cognito, IAM Identity Center, AWS KMS, AWS Certificate Manager, Parameter Store, and Secrets Manager](1.7-week7/)

**Week 8:** [Learning the AWS Solutions Architect mindset, the six pillars of the AWS Well-Architected Framework, highly available architecture, cost optimization, and architecture diagram design](1.8-week8/)

**Week 9:** [Reviewing and systematizing AWS knowledge, comparing EC2-based and serverless architectures, creating a service mind map, and developing an AWS Associate certification learning roadmap](1.9-week9/)

**Week 10:** [Completing the final AWS assessment, preparing a technical proposal, creating an architecture diagram, and estimating operating costs with AWS Pricing Calculator](1.10-week10/)

**Week 11:** [Building and securing a serverless API using Amazon Cognito, API Gateway, AWS Lambda, AWS WAF, IAM permissions, and Amazon S3 Presigned URLs](1.11-week11/)

**Week 12:** [Developing an event-driven image-processing workflow using Amazon S3, AWS Lambda, DynamoDB Streams, Amazon Rekognition, and Amazon CloudWatch](1.12-week12/)

Through these twelve weeks, I improved my understanding of AWS infrastructure, networking, computing, storage, databases, data analytics, cloud security, serverless architecture, monitoring, and cost management.

I also strengthened my practical skills in architecture design, troubleshooting, technical documentation, independent learning, resource management, and project organization. At the end of the internship, I completed the project documentation and published the complete source-code repository with a detailed `README.md` installation guide on my personal GitHub account.
